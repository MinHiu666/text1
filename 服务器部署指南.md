# 网站服务器部署指南

## 一、服务器购买

### 1.1 选择云服务商
推荐选择以下主流云服务商之一：
- **阿里云**：https://www.aliyun.com/
- **腾讯云**：https://cloud.tencent.com/
- **华为云**：https://www.huaweicloud.com/

### 1.2 选择服务器配置
| 配置项 | 推荐配置 | 说明 |
|-------|---------|------|
| CPU | 1核 | 入门级足够 |
| 内存 | 2GB | 建议至少2GB |
| 存储 | 40GB SSD | SSD硬盘速度更快 |
| 带宽 | 1-2M | 根据访问量调整 |
| 地区 | 就近原则 | 选择离目标用户近的区域 |

### 1.3 选择操作系统
- **推荐**：Ubuntu 22.04 LTS
- **备选**：CentOS 7 / CentOS Stream

## 二、服务器基础配置

### 2.1 登录服务器
使用SSH工具登录（Windows推荐Putty或Xshell，Mac/Linux直接用终端）：
```bash
ssh root@你的服务器IP地址
```

### 2.2 更新系统
```bash
# Ubuntu/Debian
apt update && apt upgrade -y

# CentOS
yum update -y
```

### 2.3 配置安全组/防火墙
开放必要端口：
- **80端口**：HTTP访问
- **443端口**：HTTPS访问
- **22端口**：SSH登录

在云服务商控制台配置安全组规则。

## 三、安装Node.js

### 3.1 安装Node.js（Ubuntu/Debian）
```bash
# 安装Node.js 20.x
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证安装
node -v  # 显示版本号即可
npm -v
```

### 3.2 安装Node.js（CentOS）
```bash
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs

# 验证安装
node -v
npm -v
```

## 四、部署网站代码

### 4.1 创建网站目录
```bash
mkdir -p /var/www/yourdomain.com
cd /var/www/yourdomain.com
```

### 4.2 上传代码
将本地项目打包后上传到服务器：
```bash
# 在本地项目目录执行
npm run build
# 将dist文件夹上传到服务器
```

或者使用Git克隆（如果有代码仓库）：
```bash
git clone https://你的代码仓库地址.git .
npm install
npm run build
```

## 五、安装并配置Nginx

### 5.1 安装Nginx
```bash
# Ubuntu/Debian
apt install nginx -y

# CentOS
yum install nginx -y
```

### 5.2 配置Nginx
```bash
# 创建站点配置文件
nano /etc/nginx/sites-available/yourdomain.com

# 添加以下内容：
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    root /var/www/yourdomain.com/dist;
    index index.html index.htm;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### 5.3 启用站点
```bash
# 创建软链接
ln -s /etc/nginx/sites-available/yourdomain.com /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重启Nginx
systemctl restart nginx
```

## 六、配置HTTPS（SSL证书）

### 6.1 安装Certbot
```bash
# Ubuntu/Debian
apt install certbot python3-certbot-nginx -y

# CentOS
yum install certbot python3-certbot-nginx -y
```

### 6.2 获取免费SSL证书
```bash
certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

按照提示完成配置，Certbot会自动配置Nginx。

### 6.3 自动续期证书
```bash
# 测试自动续期
certbot renew --dry-run

# 设置定时任务（通常已自动配置）
crontab -e
# 添加：0 12 * * * /usr/bin/certbot renew --quiet
```

## 七、域名配置

### 7.1 购买域名
在域名注册商购买域名：
- **阿里云域名**：https://wanwang.aliyun.com/
- **腾讯云域名**：https://dnspod.cloud.tencent.com/
- **GoDaddy**：https://www.godaddy.com/

### 7.2 域名备案（国内服务器）
如果使用国内服务器，必须先备案：
1. 在云服务商控制台提交备案申请
2. 准备材料：营业执照、法人身份证、域名证书
3. 等待审核（约10-20个工作日）

### 7.3 配置DNS解析
在域名管理后台添加解析记录：
| 记录类型 | 主机记录 | 记录值 | TTL |
|---------|---------|-------|-----|
| A | @ | 你的服务器IP | 5分钟 |
| A | www | 你的服务器IP | 5分钟 |

## 九、常见问题排查

### 9.1 网站无法访问
1. 检查服务器防火墙/安全组是否开放80/443端口
2. 检查Nginx配置是否正确：`nginx -t`
3. 检查Nginx日志：`cat /var/log/nginx/error.log`

### 9.2 SSL证书问题
1. 检查域名解析是否正确
2. 检查证书是否过期：`certbot certificates`
3. 手动续期：`certbot renew`

### 9.3 页面显示异常
1. 检查dist目录是否上传完整
2. 检查Nginx配置的root路径是否正确
3. 检查浏览器控制台是否有错误

## 十、维护建议

### 10.1 定期更新系统
```bash
# 定期执行
apt update && apt upgrade -y  # Ubuntu
yum update -y  # CentOS
```

### 10.2 定期备份
```bash
# 备份网站文件
tar -czvf backup_$(date +%Y%m%d).tar.gz /var/www/yourdomain.com

# 备份Nginx配置
tar -czvf nginx_backup_$(date +%Y%m%d).tar.gz /etc/nginx
```

### 10.3 监控网站状态
```bash
# 查看Nginx状态
systemctl status nginx

# 查看访问日志
tail -f /var/log/nginx/access.log
```
